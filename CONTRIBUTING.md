# Contributing

## Branching
- `main` for stable snapshots
- feature branches for module work

## Suggested commit scope
- `feat:` new features
- `fix:` bug fixes
- `refactor:` structural improvements
- `docs:` documentation updates

## Before opening a PR
- Run PHP syntax checks.
- Verify installer and login still work.
- Verify company scoping is preserved on affected records.
- Document schema changes in `database/schema.sql`.
